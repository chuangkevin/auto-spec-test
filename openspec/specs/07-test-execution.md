# 功能模組：測試執行（內嵌瀏覽器即時視覺化）

## 設計概念

系統內嵌瀏覽器畫面，AI Agent 逐條執行測試案例，使用者可在介面上即時觀看操作過程。
支援兩種來源的測試：

- **從專案發起**：使用已產出的測試腳本
- **從 URL 快速測試發起**：AI 自動掃描頁面產出的任務清單

技術方案：後端 Playwright 操作瀏覽器，透過 WebSocket 即時串流截圖到前端顯示（約 2-4 fps）。

## 發起測試設定

### 從專案內發起

| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 目標網址 | URL input | 是 | 驗證 URL 格式，測試前做連線檢查 |
| 測試帳號 | text | 否 | 目標網站的登入帳號 |
| 測試密碼 | password | 否 | 目標網站的登入密碼 |
| 執行範圍 | checkbox group | 否 | 可勾選/取消勾選要執行的測試案例，預設全部勾選 |
| 執行順序 | drag & drop | 否 | 可拖曳調整測試案例的執行順序 |
| 執行模式 | radio | 否 | 自動模式（預設）/ 逐步模式（每案例完成後暫停等待確認） |

自動使用該專案最新版的測試腳本。

## 任務清單互動

使用者在測試開始前及測試過程中可進行以下操作：

### 測試前

| 操作 | 說明 |
|------|------|
| 勾選/取消勾選 | 決定哪些測試案例要執行 |
| 拖曳排序 | 調整測試案例的執行順序 |
| 新增案例 | 手動輸入測試案例名稱、步驟、預期結果 |
| 編輯案例 | 修改現有案例的步驟或預期結果 |
| 刪除案例 | 移除不需要的案例（僅限手動新增的） |

### 測試中

| 操作 | 說明 |
|------|------|
| 勾選/取消勾選待執行案例 | 即時調整後續要測試的範圍 |
| 新增案例 | 測試過程中發現新的測試情境，即時新增到清單尾端 |
| 調整待執行案例順序 | 已執行的案例順序鎖定，僅待執行的可調整 |

## 手動介入模式

當使用者點擊「手動介入」或系統處於暫停狀態時：

1. AI 暫停操作，瀏覽器保持當前狀態
2. 前端切換為「手動操作提示」，告知使用者目前可以手動操作
3. 使用者在系統內看到即時畫面，但此時由使用者在目標網站上手動操作（例如：處理驗證碼、登入第三方帳號等 AI 無法處理的步驟）
4. 使用者完成手動操作後，點擊「繼續」，AI 從當前頁面狀態繼續執行

**手動介入的場景：**

- AI 無法處理的操作（如：圖形驗證碼、兩步驟驗證）
- 使用者想手動設定特定資料或狀態
- 使用者想觀察特定頁面後再繼續

## 執行流程（逐案例）

1. 後端建立 Playwright Browser Context + WebSocket 串流截圖
2. AI Agent 依照任務清單的順序，取得下一個已勾選的案例
3. 逐步驟執行（截圖+DOM → AI 判斷 → 操作 → 下一步）
4. 每步驟截圖即時推送前端
5. 案例完成後更新狀態（綠燈/紅燈）
6. 自動模式：直接進入下一案例 / 逐步模式：暫停等待確認
7. 全部完成後產出測試報告

## 即時互動控制

| 操作 | 說明 |
|------|------|
| 暫停 | 暫停測試，瀏覽器保持當前狀態 |
| 繼續 | 從暫停處繼續 |
| 手動介入 | 暫停 AI，使用者手動操作瀏覽器，完成後繼續 |
| 跳過此案例 | 標記為 skip，進入下一個 |
| 終止測試 | 結束測試，已完成案例仍產出報告 |
| 重測此案例 | 對失敗案例重新執行 |
| 標記已知問題 | 手動標記，不列入 Bug 清單 |

## 案例即時狀態（燈號）

| 狀態 | 燈號 | 說明 |
|------|------|------|
| 待執行 | ⚪ 灰燈 | 尚未輪到 |
| 執行中 | 🟡 黃燈 | 目前正在執行（高亮動畫） |
| 通過 | 🟢 綠燈 | 測試通過 |
| 失敗 | 🔴 紅燈 | 測試失敗（點擊可查看截圖與原因） |
| 跳過 | ⏭ 灰燈 | 被使用者跳過 |
| 已知問題 | 🟠 橙燈 | 手動標記的已知問題 |

## 截圖串流規格

| 項目 | 規格 |
|------|------|
| 串流方式 | WebSocket base64 JPEG |
| 更新頻率 | 2-4 fps（操作時 4 fps，靜止時 1 fps） |
| 解析度 | 1280x720 |
| JPEG 品質 | 60%（串流）/ 90%（關鍵截圖保存） |
| 每幀大小 | 約 50-150 KB |

## 測試限制

| 項目 | 限制 |
|------|------|
| 單一案例超時 | 60 秒 |
| 整體測試超時 | 30 分鐘 |
| 同時執行數量 | 3 個（可調整） |
| 瀏覽器視窗 | 預設 1280x720 |
