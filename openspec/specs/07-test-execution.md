# 功能模組：測試執行（內嵌瀏覽器即時視覺化）

## 設計概念
系統內嵌瀏覽器畫面，AI Agent 逐條執行測試案例，使用者可在介面上即時觀看操作過程。

技術方案：後端 Playwright 操作瀏覽器，透過 WebSocket 即時串流截圖到前端顯示（約 2-4 fps）。

## 測試執行頁面佈局

```text
┌─────────────────────────────────────────────────────────────┐
│  測試設定列：[目標網址 ___________] [測試帳號] [測試密碼]       │
│             [執行模式: ◉自動 ○逐步] [▶ 開始測試]              │
├────────────────────────────────┬────────────────────────────┤
│                                │  測試案例清單               │
│     內嵌瀏覽器畫面              │  TC-001 登入功能    ✅      │
│     (WebSocket 截圖串流)        │  TC-002 首頁顯示    ▶🔄    │
│                                │  TC-003 搜尋功能    ⏳      │
│     ┌────────────────────┐    │                            │
│     │   目標網站即時畫面   │    │  目前執行步驟               │
│     │                    │    │  步驟 2/4: 在「帳號」        │
│     └────────────────────┘    │  欄位輸入 test@mail.com     │
│                                │  AI 判斷：欄位已找到...      │
├────────────────────────────────┴────────────────────────────┤
│  操作列：[⏸ 暫停] [▶ 繼續] [⏭ 跳過此案例] [⏹ 終止測試]      │
│  即時 Log                                                    │
└─────────────────────────────────────────────────────────────┘
```

## 發起測試設定

### 從專案內發起
| 欄位 | 類型 | 必填 | 說明 |
|------|------|------|------|
| 目標網址 | URL input | 是 | 驗證 URL 格式，測試前做連線檢查 |
| 測試帳號 | text | 否 | 目標網站的登入帳號 |
| 測試密碼 | password | 否 | 目標網站的登入密碼 |
| 執行範圍 | checkbox group | 否 | 可選擇特定測試案例，預設全部執行 |
| 執行模式 | radio | 否 | 自動模式（預設）/ 逐步模式（每案例完成後暫停等待確認） |

### 從「系統測試」入口發起
- 輸入目標網址
- 選擇腳本來源：從專案選取 / 上傳腳本（.md）

## 執行流程（逐案例）
1. 後端建立 Playwright Browser Context + WebSocket 串流截圖
2. AI Agent 取得測試案例，逐步驟執行（截圖+DOM → AI 判斷 → 操作 → 下一步）
3. 每步驟截圖即時推送前端
4. 自動模式：直接進入下一案例 / 逐步模式：暫停等待確認
5. 全部完成後產出測試報告

## 即時互動控制
| 操作 | 說明 |
|------|------|
| 暫停 | 暫停測試，瀏覽器保持當前狀態 |
| 繼續 | 從暫停處繼續 |
| 跳過此案例 | 標記為 skip，進入下一個 |
| 終止測試 | 結束測試，已完成案例仍產出報告 |
| 重測此案例 | 對失敗案例重新執行 |
| 標記已知問題 | 手動標記，不列入 Bug 清單 |

## 案例即時狀態
| 狀態 | 說明 |
|------|------|
| ⏳ 待執行 | 尚未輪到 |
| ▶🔄 執行中 | 目前正在執行（高亮） |
| ✅ PASS | 測試通過 |
| ❌ FAIL | 測試失敗（點擊可查看截圖與原因） |
| ⏭ SKIP | 被使用者跳過 |
| ⚠️ 已知問題 | 手動標記的已知問題 |

## WebSocket 協議
```jsonc
// Client → Server:
{ "type": "start_test", "payload": { "executionId": "...", "targetUrl": "..." } }
{ "type": "pause" }
{ "type": "resume" }
{ "type": "skip_case" }
{ "type": "stop_test" }
{ "type": "retry_case", "payload": { "testCaseId": "TC-001" } }

// Server → Client:
{ "type": "screenshot", "payload": { "base64Image": "...", "timestamp": 1234 } }
{ "type": "case_start", "payload": { "testCaseId": "TC-001", "index": 1, "total": 10 } }
{ "type": "step_update", "payload": { "testCaseId": "TC-001", "stepIndex": 2, "description": "..." } }
{ "type": "case_result", "payload": { "testCaseId": "TC-001", "status": "pass" } }
{ "type": "test_complete", "payload": { "reportId": 1, "summary": {} } }
```

## 截圖串流規格
| 項目 | 規格 |
|------|------|
| 串流方式 | WebSocket base64 JPEG |
| 更新頻率 | 2-4 fps（操作時 4 fps，靜止時 1 fps） |
| 解析度 | 1280x720 |
| JPEG 品質 | 60%（串流）/ 90%（關鍵截圖保存） |
| 每幀大小 | 約 50-150 KB |

## 測試限制
| 項目 | 限制 |
|------|------|
| 單一案例超時 | 60 秒 |
| 整體測試超時 | 30 分鐘 |
| 同時執行數量 | 3 個（可調整） |
| 瀏覽器視窗 | 預設 1280x720 |
